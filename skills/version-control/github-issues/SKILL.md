# GitHub Issues

Use this skill for github issues work in Git or GitHub repositories.

## When to activate

- User asks to file a bug, feature request, task, or update an issue.
- Need issue search, labels, assignees, milestones, relationships, or project fields.
- Need to format issues from repository templates.

## Workflow

1. Identify repo, issue number or creation intent, and available templates/labels.
2. Search for duplicates before creating new issues when appropriate.
3. Apply labels, assignees, milestones, issue type, dependencies, and project fields requested.
4. Return issue URLs and any fields that could not be set.

## Safety rules

- Do not create or modify issues without clear user intent.
- Do not include secrets, private logs, or credentials in issue content.
