# Visual Design Foundations Full Guidance

This reference preserves the long-form guidance layer required by the update plan while the active `SKILL.md` remains a concise routing entrypoint. Use the more focused companion references for deeper examples and detailed procedures.

## Detailed guidance

- Start with product goals, brand constraints, audience needs, and accessibility baseline before choosing visual treatments.
- Use a small token vocabulary for type, color, spacing, elevation, radii, and icon sizes; document semantic intent and allowed exceptions.
- Validate hierarchy by scanning primary actions, content grouping, contrast, line length, rhythm, density, and empty/error states.
- Check dark mode, high contrast, localization, responsive type, and component reuse before finalizing visual decisions.
