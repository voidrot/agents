# Visual Design Foundations

Use this skill for focused design work in this task family. Keep the active guidance concise, and open supporting references only when their detail is needed.

## When to use

- Defining typography, color, spacing, radius, shadow, or iconography systems.
- Improving visual hierarchy, readability, consistency, or brand expression.
- Auditing a UI for visual polish, accessible contrast, dark mode, or theming readiness.

## Workflow

1. Identify the product context, brand constraints, target platforms, and accessibility baseline.
2. Inventory existing visual primitives and find gaps or inconsistent usage.
3. Choose or refine typography, color, spacing, and iconography decisions as a coherent system.
4. Specify implementation-ready tokens, component guidance, and validation checks.
5. Document trade-offs and any follow-up visual QA needed.

## Constraints

- Do not optimize visual novelty at the expense of readability, contrast, or task completion.
- Keep token names semantic enough for reuse across light/dark themes and product surfaces.
- Prefer small, composable visual decisions over one-off component styling.
