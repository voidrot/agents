# Design System Patterns Full Guidance

This reference preserves the long-form guidance layer required by the update plan while the active `SKILL.md` remains a concise routing entrypoint. Use the more focused companion references for deeper examples and detailed procedures.

## Detailed guidance

- Separate primitive, semantic, and component tokens; document names, ownership, lifecycle, and migration rules.
- Define component APIs around variants, states, accessibility contracts, composition slots, and expected design-tool mapping.
- Design theme architecture for brand, mode, density, platform, and runtime needs without over-abstracting early.
- Govern changes through versioning, contribution guidelines, documentation, visual regression checks, and adoption metrics.
