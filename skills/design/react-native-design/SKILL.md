# React Native Design

Use this skill for focused design work in this task family. Keep the active guidance concise, and open supporting references only when their detail is needed.

## When to use

- Building or reviewing cross-platform React Native layouts and styling architecture.
- Implementing navigation, gestures, animation, and transition patterns for mobile flows.
- Improving platform-specific behavior, performance, accessibility, or native quality.

## Workflow

1. Confirm app architecture, React Native version, navigation stack, animation library, and target platforms.
2. Map UI requirements to platform conventions, layout constraints, accessibility, and performance risks.
3. Choose styling, navigation, gesture, and motion patterns that preserve native responsiveness.
4. Check re-render, layout, gesture conflict, safe-area, keyboard, and reduced-motion behavior.
5. Return concrete implementation steps and testing notes for iOS and Android.

## Constraints

- Do not add animation or gesture complexity without a clear user-feedback or navigation purpose.
- Respect platform conventions where iOS and Android expectations diverge.
- Account for safe areas, keyboard behavior, accessibility labels/actions, and reduced motion.
