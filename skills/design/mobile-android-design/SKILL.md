# Android Mobile Design

Use this skill for focused design work in this task family. Keep the active guidance concise, and open supporting references only when their detail is needed.

## When to use

- Designing Android app flows with Material Design 3 and platform conventions.
- Implementing Jetpack Compose UI, theming, adaptive layout, or Navigation Compose.
- Checking Android accessibility, dynamic color, foldables/tablets, and interaction behavior.

## Workflow

1. Identify target devices, OS constraints, user tasks, navigation model, and Material 3 requirements.
2. Choose Compose components, layout patterns, and Material tokens for the hierarchy and state model.
3. Check accessibility, touch targets, text scaling, contrast, dynamic color, edge-to-edge, and adaptive layouts.
4. Specify implementation-ready composables, theme tokens, navigation behavior, and state guidance.
5. List emulator/device checks for phones, tablets, foldables, dark mode, and TalkBack paths.

## Constraints

- Prefer Material 3 and platform components before custom controls.
- Design for configuration changes, text scaling, multiple window sizes, and foldable postures.
- Do not ignore TalkBack labels, semantics, focus order, or minimum target sizes.
