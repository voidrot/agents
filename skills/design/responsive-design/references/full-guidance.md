# Responsive Design Full Guidance

This reference preserves the long-form guidance layer required by the update plan while the active `SKILL.md` remains a concise routing entrypoint. Use the more focused companion references for deeper examples and detailed procedures.

## Detailed guidance

- Design from content and user tasks rather than device names; pick breakpoints where layout or readability actually changes.
- Use container queries for reusable components and global breakpoints for page-shell or navigation shifts.
- Prefer fluid type, fluid spacing, flexible media, and CSS Grid/Flexbox patterns that preserve zoom and localization.
- Validate overflow, keyboard/focus order, touch targets, responsive tables, navigation, and reduced-motion behavior across viewport ranges.
