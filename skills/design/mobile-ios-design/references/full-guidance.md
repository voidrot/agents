# iOS Mobile Design Full Guidance

This reference preserves the long-form guidance layer required by the update plan while the active `SKILL.md` remains a concise routing entrypoint. Use the more focused companion references for deeper examples and detailed procedures.

## Detailed guidance

- Prefer Apple platform conventions, system controls, SF Symbols, Dynamic Type, and native navigation patterns.
- Design hierarchy around task flow, content priority, navigation depth, and presentation style: push, sheet, popover, tab, or split view.
- Validate VoiceOver, Dynamic Type, contrast, reduced motion, safe areas, keyboard paths, localization, dark mode, and iPad adaptation.
- Use custom components only when system components cannot meet the product need, and document the accessibility contract.
