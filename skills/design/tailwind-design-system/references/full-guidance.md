# Tailwind Design System Full Guidance

This reference preserves the long-form guidance layer required by the update plan while the active `SKILL.md` remains a concise routing entrypoint. Use the more focused companion references for deeper examples and detailed procedures.

## Detailed guidance

- Confirm the project is using Tailwind CSS v4 before relying on CSS-first configuration and v4-specific behavior.
- Model primitives as CSS variables, expose semantic tokens for components, and document allowed utility composition patterns.
- Use component variants for repeated states and sizes; avoid copying long utility strings across the codebase.
- Validate content scanning, build integration, dark mode, responsive behavior, and accessibility states after token changes.
